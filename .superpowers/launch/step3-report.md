## Exact sequence for the user

**(a) One-time key setup:**
```bash
npm run tauri signer generate -- -w ~/.tauri/kaestral-updater.key
```
- Prompts for a password (save it — needed for every future signed build as `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`).
- Prints the **public key** — paste it into `src-tauri/tauri.conf.json` → `plugins.updater.pubkey`, replacing the placeholder string, then commit (public key is safe to commit).
- Keep `~/.tauri/kaestral-updater.key` secret, never commit it.
- Before any build that should produce installable update artifacts, set:
  ```bash
  export TAURI_SIGNING_PRIVATE_KEY="$HOME/.tauri/kaestral-updater.key"   # path or raw contents
  export TAURI_SIGNING_PRIVATE_KEY_PASSWORD="<the password you set>"
  ```
  (PowerShell: `$env:TAURI_SIGNING_PRIVATE_KEY = "..."`, `$env:TAURI_SIGNING_PRIVATE_KEY_PASSWORD = "..."`.)

**(b) Publishing v1.1.0 (repeat per release):**
1. Bump version in `package.json`, `src-tauri/tauri.conf.json`, `src-tauri/Cargo.toml` → `1.1.0`.
2. With signing env vars set: `npm run tauri build` → produces `Kaestral_1.1.0_x64-setup.exe` + `.exe.sig` under `src-tauri/target/release/bundle/nsis/`.
3. Create a GitHub release tagged `v1.1.0` on `prabindersinghh/Kaestral-pro`; upload the `.exe`.
4. `node scripts/make-latest-json.mjs 1.1.0 src-tauri/target/release/bundle/nsis/Kaestral_1.1.0_x64-setup.exe.sig` → writes `latest.json`.
5. Upload `latest.json` to that same `v1.1.0` release (must be marked "latest", not draft/pre-release).
6. Existing apps' "Check for updates" button (or a future auto-check-on-launch) hits `releases/latest/download/latest.json`, verifies the signature against the baked-in pubkey, downloads, installs, and relaunches.

Full detail (including troubleshooting) is in `docs/UPDATER-SETUP.md`.

## Flags / things to note
